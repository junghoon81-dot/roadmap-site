# 김과외 5월 로드맵

> 5월 1주차~4주차(4주) 실행 범위.

**특집: 온플랫폼 관계 지속**

매칭 이후 끊기는 학부모와의 관계를 매주 체크인·코칭 리포트·중재·재매칭으로 유지한다. 관계가 플랫폼 안에 남으면 직거래 누수가 줄고, 거래량이 누적된다.

> **실행:** 5월 1주차~4주차(4주). 1~2주차 팀 진행, 현진님 복귀 후 본격 개편 개발.

---

## 1. 왜 매칭 이후가 핵심인가

### 김과외의 구조적 문제

김과외의 거래량은 정체되어 있습니다. 원인은 세 가지가 맞물려 있습니다.

**첫째, 매칭이 성사되면 관계가 끊긴다.** 선생님과 학부모가 카톡으로 소통하고 계좌이체로 결제하면 김과외가 끼어들 자리가 없습니다. 수업이 잘 되고 있는지, 문제가 있는지, 끝났는지조차 모릅니다.

**둘째, 직거래로 빠진다.** 다음에 과외를 다시 구할 때 김과외로 돌아오기도 하지만, 선생님 직접 추천이나 맘카페를 통한 직거래로 빠지는 경우가 많습니다.

**셋째, 시장 자체가 안 큰다.** 출산율 하락으로 학생 수가 줄어드는 추세에서, 첫 매칭 성사 한 번만으로는 거래량 성장에 한계가 있습니다.

이 세 가지를 관통하는 핵심은 **"매칭 이후 김과외의 역할이 없다"**는 것입니다.

### 이번 패치의 핵심

매칭 성사 후 매주 가벼운 체크인으로 관계를 끊지 않고, 이상 신호가 감지되면 선생님에게 코칭 리포트를 전달하고, 조정이 안 되면 재매칭으로 연결합니다.

## 2. 매출 연결 논리

### 매출 공식 — 실매출 vs 운영 드라이버 분리

기존 `매출 = 신규 성사건수 × 온플랫폼 유지개월 × 월 take rate`는 실제 수익 모델(1회 매칭 수수료)과 미정합합니다. 유지개월은 매출의 직접 항이 아니라 간접 드라이버입니다.

**실매출 공식:** `실매출 = 신규 유료 성사 × 평균 수수료 + 재매칭 유료 성사 × 평균 수수료 + 확장(추가 과목/형제) 유료 성사 × 평균 수수료`

**운영 드라이버 공식:** `유지개월`은 재매칭·확장·재방문 확률을 높이는 **선행지표**로 위치시킵니다. 체크인이 잘 돌아가면 유지개월이 늘고, 유지개월이 늘면 재매칭/확장/추천 확률이 올라가고, 이것이 실매출의 세 항목을 움직입니다.

### KPI 3층 구조

KPI 단위가 섞이면 해석이 틀어집니다. 최소한 `match_id`, `student_id`, `household_id`, `teacher_id` 4개 단위를 분리합니다.

**비즈니스 KPI:** 신규 유료 성사, 재매칭 유료 성사, 확장 유료 성사, repeat-household revenue

**운영 KPI:** 체크인 응답률, status observability rate, 문제 감지율, 개입 수락률, 재매칭 제안 수락률, 프로필 핵심필드 커버리지

**데이터 무결성 KPI:** 종료사유 포착률, unknown 비율, source mix, label agreement rate

### "온플랫폼 지속" 용어 교정

카톡/계좌이체로 플랫폼 밖에서 수업이 진행되므로, 체크인 응답이 왔다고 "플랫폼 안"이라 할 수 없습니다. KPI 명칭을 `온플랫폼 지속` → **`김과외 관여 지속`** 또는 **`관측 가능 유지기간`**으로 교정합니다. 진짜 `온플랫폼`은 결제/정산/수업관리까지 플랫폼이 통제하는 경우에만 사용합니다.

### 유지기간 측정 4상태 모델

"수업이 아직 진행 중" 상태가 DB에 없고, 체크인 응답률 10~15%이면 나머지 85~90%는 미관측입니다. 미응답은 랜덤 누락이 아니라 만족/불만족/무관심이 뒤섞인 편향 표본입니다.

**4상태 모델:** `confirmed_active` / `confirmed_ended` / `inferred_active` / `unknown`

`status_observability_rate` = (confirmed_active + confirmed_ended) / 전체 활성 매칭. 이 비율이 40% 미만이면 유지율 KPI를 대외 주장하지 않습니다.

### 자연 이탈 vs 방지 가능한 이탈 — 3단계 분류

이진 분류 대신 `avoidable / mixed / unavoidable` 3단계로 나눕니다. "비용 부담"도 "예상보다 비싸서 불만"은 방지 가능, "가계 사정 악화"는 자연에 가깝습니다.

### 성수기 타이밍 — 선행지표 중심

7월 초 매칭이 들어와도 첫 체크인→코칭→회복 관찰까지 최소 2~3주. 7월 KPI를 `발송성공률, 응답률, 문제감지율, 개입률` 선행지표로 두고, 유지/재매칭 효과는 8~10월 코호트로 측정합니다. 6월 하순에 5~10% 코호트 소규모 파일럿을 먼저 돌려 템플릿·선택지·에스컬레이션 룰을 다듬습니다.

### ① 매주 체크인 + 코칭 리포트 + 중재 — 상태 머신

#### Stage 1 — Pulse (매주 체크인)

매주 알림톡 1통. "이번 주 수업 어떠셨어요?" → 😊 좋아요 / 😐 보통 / 😟 아쉬워요 (3단계). 탭 1번이면 끝.

- 😊 → 끝. 추가 질문 없음.
- 😐 → 추가 질문: "어떤 부분이 아쉬우셨나요?" → 선택지
- 😟 → 같은 추가 질문 + "김과외에서 도움을 드릴까요?" → 3개 선택지

**선택지 (8개):** 수업 내용/설명 | 관리/숙제 | 일정/시간 | 소통 | 비용 | 핏/분위기 | 아직 판단 어려움 | 기타

**gray(무응답/판단불가) 상태:** `😊 3회`만으로 그린 전환하지 않습니다. `무응답 없음 + 고정 시점 설문 이상 없음 + 선생님 플래그 없음`까지 함께 확인합니다.

**상태 모델 — 4색 + 케이스 스테이지 분리:**

`health color`와 `case stage`를 별도 필드로 분리합니다.

| health color | 조건 | 체크인 빈도 |
|-------------|------|------------|
| **green** | 😊 3회 연속 + 무응답 없음 + 선생님 플래그 없음 | 격주 |
| **yellow** | 😐 또는 😟 1회 | 매주 |
| **red** | 😟 2주 연속 또는 동일 카테고리 😐 3회 | 매주 + Stage 3 전환 |
| **gray** | 무응답 2회 이상 또는 정보 부족 | 별도 처리 |

**심각도별 전환 규칙 차등화:** 모든 문제를 같은 잣대로 보지 않습니다.

| 심각도 | 유형 예시 | 전환 규칙 |
|--------|----------|----------|
| 즉시 Stage3 | 무례, 안전, 시간 단축 | 1회 즉시 |
| 1회 재확인 | 시간 미준수, 잦은 취소 | 다음 체크인 확인 후 |
| 2회 누적 | 숙제관리, 소통 부족 | 동일 카테고리 2회 |
| 3회 관찰 | 진도 속도, 설명 스타일 | rolling risk score 누적 |

**하향/회복 규칙:** `green + 😟 = 즉시 yellow`. 하드 분쟁 카테고리면 `즉시 Stage3`. `red → green` 복귀는 1회 긍정이 아니라 **2~3회 안정 응답 후**입니다.

**rolling risk score:** 카테고리 반복은 가중치로 두고, 기본은 rolling risk score로 누적 관리합니다. "숙제 1회 + 시간 1회 + 소통 1회"도 충분히 위험 신호입니다.

**발송 타이밍:** 일요일/월요일 일괄 발송은 수업 cadence를 무시합니다. 원칙은 **지난 7일 내 마지막 수업 후 24~36시간**으로 두고, 그 주 수업이 없으면 만족도 대신 **수업 진행 여부**를 먼저 묻습니다.

**무응답 처리:** 2회 무응답 → 빈도 축소 전에 `월 1회 / 문제 있을 때만 / 계속 주 1회` preference를 먼저 수집합니다.

**다자녀/다과목 가정:** 매칭 단위 발송 시 한 가정에 주 2~4통이 갑니다. 학부모 측은 **가정 단위 주간 digest**로 묶습니다.

**opt-out 설계:** `월 1회` / `중요한 경우만` / `중단`을 명시적으로 제공합니다. 카카오 차단이 쌓이면 채널 자체가 약해집니다.

#### 코칭 리포트 — 정기 발송 + 케이스 분리

모든 활성 선생님에게 매주 발송합니다. 7가지 콘텐츠 소스에서 매번 2~3개를 조합:

| 소스 | 내용 | 언제 실리나 |
|------|------|------------|
| ① 강점 피드백 | 학부모 긍정 응답 기반 | 긍정 데이터가 있을 때 (항상 최우선 배치) |
| ② 개선 제안 | 😐/😟 카테고리 기반 | 부정 데이터가 있을 때만 |
| ③ 과목/학년별 수업 팁 | 과목×학년×시기×수업단계 조합형 템플릿 | 매주 (빈 리포트 방지) |
| ④ 전체 통계/벤치마크 | 수업 유지율 순위, 평균 만족도 | 데이터 충분할 때 |
| ⑤ 시즌/시기별 안내 | 중간고사, 방학 등 | 해당 시기에 |
| ⑥ 프로필/응답률 넛지 | 완성도, 응답 속도 | 빈 프로필 또는 응답 개선 여지 시 |
| ⑦ 학부모 총평 | 최근 3개월 N명 기준 강점/보완 | Phase 2 이후 |

**팁 콘텐츠 고갈 방지:** 정적 팁 50~100개 로테이션은 26주면 반복 체감. `과목 × 학년 × 시기 × 수업 단계 × 위험 상태` **조합형 템플릿**으로 설계합니다.

**정기 리포트 vs 케이스 개입 분리:** 레드 리포트의 `중요` 표시와 정기 프레이밍이 충돌합니다. **정기 코칭 리포트 = 비케이스성**, **경고/시정요청 = CS 발신 별도 커뮤니케이션**으로 채널을 나눕니다. 하드 분쟁은 정기 리포트 원칙 바깥으로 빼고 즉시 CS 플로로 보냅니다.

**초기 리포트 유통기한:** 2주차에도 "새 수업이 시작되었습니다"가 나오면 어색합니다. `첫 1회만 새 수업`, `2~4주 초기 적응기`, 이후 일반 리포트로 단계명을 나눕니다.

**익명성 강화:** 학생 1명인 선생님은 출처를 추정할 수 있습니다. 표현을 `최근 수업 운영 흐름상` 수준으로 낮추고, `직접 인용 금지`, `정확한 시점 노출 금지`, `희귀 표현 제거`, **N이 작은 경우 집계형 문구 금지** 가드레일을 적용합니다. 활성 학생 수가 적은 선생님에게는 부정 피드백을 자동 리포트에 직접 싣지 말고, 일반 팁만 보내거나 CS 중재 경로로 우회합니다.

**리포트 효과 측정:** "리포트 후 2주 내 그린 복귀"는 자연 회복과 구분되지 않습니다. 안전 개입은 모두 하되, **개선 포인트 개인화 강도** 또는 **리포트 문안**만 랜덤 분기하여 14일 회복률과 30일 유지율을 비교합니다.

**행동 변화 루프:** 리포트에 `이번 주 이렇게 조정해보겠습니다` 1탭 확인 + 간단한 실행 선택지를 넣고, 이후 2주간 동일 카테고리 재발률로 효과를 봅니다.

**선생님 체감 가치 — "감시"가 아니라 "보호 + 회복 지원 + 강점 증폭":** 우수 선생님일수록 "나를 평가만 하고 대우는 그대로"라고 느끼면 이탈합니다. 리포트보다 먼저 가치 교환을 보여줍니다: **우선 노출**, **분쟁 시 보호**, **재매칭 우선권**, **프로필 강화**, **좋은 피드백의 외부 반영**. 그린 상태 장기 유지 선생님은 매주→월간 digest로 전환합니다.

**온보딩 문구 교정:** "데이터 기반 코칭"은 과장. → "학부모 체크인, 운영 데이터, 선생님 입력을 종합한 **참고용 리포트**"로 설명합니다. 제재용이 아니라 회복 지원용이라는 점, 단건 피드백은 바로 불이익으로 쓰지 않는다는 원칙을 명시합니다.

**게임화 방지:** 단일 체크인은 랭킹/노출에 직접 반영하지 않습니다. **반복 패턴 + 다른 신호가 맞을 때만** 활용합니다. 부모에게는 "응답은 비공개 처리"를 명확히, 선생님에게는 정확한 임계치와 점수 규칙을 노출하지 않습니다.

#### Stage 2 — Triage (심각도 판단 + Stage 3 전환)

코칭 리포트는 매주 자동 발송이므로, Stage 2의 역할은 **"개선 제안이 2주 이상 반복됐는데 호전 없으면 Stage 3으로 전환하는 판단"**입니다.

- 개선 포인트 2회 연속 + 학부모 여전히 옐로/레드 → Stage 3
- 학부모가 직접 "김과외 도움 요청" 선택 → 즉시 Stage 3
- 하드 분쟁 카테고리(무례, 안전, 시간 단축) → 1회 즉시 Stage 3

#### Stage 3 — Mediation (CS 중재)

**CS 자동 에스컬레이션:** 슬랙/CS 도구에 티켓 자동 생성. 티켓 포함: 매칭 ID, 체크인 히스토리, 불만족 카테고리, 코칭 리포트 이력, 학부모 희망 해결 방식.

**티켓 필수 입력:** `심각도`, `반복성`, `최근성`, `양측 계속 의사`, `다음 수업일`, `기존 이력`, `증빙 유무`. 결과는 `모니터링` / `개선계획` / `즉시 재매칭` / `종결` 4분기 고정.

**CS 스크립트 표준화:** "재매칭 드릴까요?"는 실질적으로 종료 유도일 수 있습니다. `계속 + 조정` / `병행 탐색` / `종료 후 재매칭` 3옵션 고정형으로 표준화합니다.

**선생님 rebuttal 창구:** Stage 3에서는 선생님에게도 짧은 사실확인 폼을 열고, 그 답변이 같은 티켓에 남게 합니다.

**CS 권한 매트릭스:**

| 가능 | 불가 |
|------|------|
| 사실 확인 (양쪽 입장 청취) | 개인 계약 환불 강제 |
| 피드백 전달 (양방향) | 수업료 강제 조정 |
| 조정안 제시 | 수업 종료 강제 |
| 재매칭 우선 지원 | 일방적 선생님 제재 |
| 수업 기록 정리 | 법적 분쟁 중재 |

#### Stage 4 — Rematch (재매칭)

재매칭 전환 상태: `exploring_rematch → transition_pending → ended`. 기존 수업 종료 전에 학부모에게 `계속 진행 중 / 일시 중단 원함 / 이미 종료함`을 먼저 받습니다.

- 기존 수업 데이터(불만족 사유, 체크인 히스토리)를 반영하여 같은 실수 반복 방지
- 재매칭 성사 후에도 동일한 매주 체크인 파이프라인 가동

#### 선생님 양방향 설계

**선생님 체크인 (월 1회):** "이번 달 수업 진행에 어려움은 없으셨나요?"

**선생님 과도한 요구 taxonomy:** `업무 범위 확대` / `오프아워 응답 압박` / `무상 추가 업무` / `잦은 일정 변경` / `직거래 압박` / `무례/폭언`으로 정책형 taxonomy를 구성합니다. 각 항목마다 대응 매뉴얼을 붙이고, 선생님에게도 `기록만` / `템플릿 답변 요청` / `CS 개입 요청` 세 단계를 제공합니다.

**상시 창구:** 월간 정기 체크인은 유지하되, 매주 리포트마다 `이번 주 어려움 있음` **1탭 신고**를 붙여 상시 창구를 둡니다.

**직거래 압박 후속 액션:** `가정 risk tag`, `반복 시 CS 직접 안내`, `선생님용 응답 템플릿`, `플랫폼 통해야 지원 가능한 범위 명시`까지 붙여야 의미가 생깁니다.

#### 이벤트 기반 고정 시점 설문

매주 체크인은 가벼운 상태 확인, 고정 시점 설문은 깊은 데이터 수집. 같은 주에 겹치면 고정 시점 설문만 보내고 체크인은 스킵.

고정 시점 설문도 첫 화면은 동일한 **1탭 Pulse**로 시작하고, 응답 후에만 상세 질문을 엽니다. `2주 이상 무응답`을 종료 감지에 쓰면 오탐이 큽니다 — **무응답 + 수업 미진행 신호 + 선생님 확인** 2개 이상 신호가 겹칠 때만 종료 설문을 보냅니다.

| 시점 | 질문 핵심 | 트리거 |
|------|-----------|--------|
| 첫 수업 시작 확인 | 수업 시작 여부 + 첫인상 | 첫 수업일 확인 후 3일 |
| 수업 2~3회 후 | 만족도 + 강점 + 지속 의향 | 첫 수업일 + 14일 |
| 수업 1개월 후 | 만족도 + 보완점 + 목표달성도 | 첫 수업일 + 30일 |
| 수업 3개월 후 | 성적변화 + 종료이유 + NPS | 첫 수업일 + 90일 |
| 수업 종료 감지 시 | 종료 사유 | 복합 신호 기반 |

**충분조건 수치화:** 프로필은 **핵심 3필드 커버리지 20~30%** 이상이어야 검색/재매칭에서 체감 변화가 생깁니다. 추천카드는 `문의 전환율`, `재매칭 수락률`, `후보 결정까지 걸린 시간`으로 효과를 봅니다.

### ⑤ taxonomy 설계

**수집 속도 모델:** `월 4,000 eligible × 응답 10% × 문제 20% × 카테고리 선택 80%` = 체크인에서 월 ~64개 usable label. 2,500개까지 체크인 단독으로 ~39개월. **14일/30일/90일/종료 설문 + 제안거절 로그**가 합쳐져야 현실적 속도가 나옵니다.

**영끌 데이터 source weighting:** 회고 인터뷰는 기억 왜곡, 리뷰는 긍정 편향, CS는 불만 과대표집. `실시간 체크인 > 종료 설문 > CS 대화록 > 후기 텍스트 > 회고 전화` 순으로 신뢰도 차등. 영끌 데이터는 **taxonomy 시드용**, 제품 내 실시간 데이터는 **모델 학습용**으로 역할 분리합니다.

**NLP confidence threshold:** `0.85 이상 자동 승인`, `0.6~0.85만 검수`, `0.6 미만은 자유서술 보존만`. 초반에는 상위 8~10개 카테고리만 다룹니다.

**부정 리뷰 표본 확인:** 125K 리뷰 중 실제 부정/중립 비율부터 실측합니다. 부정이 2~5%면 세그먼트까지 쪼개는 순간 표본이 급감합니다. `낮은 평점 리뷰 + 종료 설문 부정 + CS 불만 텍스트 + 체크인 자유서술`을 합친 **complaint corpus**를 만들어 top-level taxonomy부터 잡습니다.

## 7. 확인 필요 사항

이 문서에서 DB 존재 여부 및 외부 공개 가능 품질이 확인되지 않은 항목:
