# 김과외 6월 로드맵 v2.0

> 6월 실행 범위. 5월 Phase 1에서 축적된 판단 데이터 위에 **판단 보조 시스템**을 완성합니다.

---

## 1. 왜 "판단 보조 시스템"인가 — 문제의식

### 학부모가 사는 건 "선생님 목록"이 아니다

학부모가 김과외에 돈을 내는 이유를 정확히 짚어야 합니다. 학부모는 선생님 프로필을 구경하려고 오는 게 아닙니다. 학부모가 진짜 사는 건 **"우리 아이에게 이 선생님이 맞을 거라는 확신"**입니다.

과외 선생님을 고르는 건 학부모 입장에서 상당히 불안한 의사결정입니다. 잘못 고르면 아이의 시간과 돈을 동시에 날립니다. 학원은 이름이라도 있고 커리큘럼이라도 정해져 있는데, 과외는 선생님 한 명에 전적으로 의존합니다. 그래서 학부모는 "가장 좋은 선생님"보다 **"실패하지 않을 선생님"**을 찾습니다.

지금 김과외가 하고 있는 건 "선생님 프로필을 보여주고, 학부모가 알아서 판단하게 하는 것"입니다. 이건 판단의 부담을 학부모에게 떠넘기는 구조입니다. 프로필을 아무리 예쁘게 만들어도, 학부모는 "이 사람이 진짜 우리 애한테 맞을까?"라는 불안을 혼자 안고 결정해야 합니다.

### "선생님 목록" vs "판단 근거"의 돈 차이

"선생님 목록을 보여주는 곳"은 대체제가 많습니다. 숨고, 크몽, 당근, 네이버 카페. 프로필 DB는 누구나 만들 수 있고, 선생님은 여러 플랫폼에 동시 등록합니다. 이 시장에서 경쟁하면 결국 광고비 싸움, 수수료 인하 싸움으로 갑니다.

반면 **"왜 이 선생님이 우리 아이에게 맞는지 근거를 대주는 곳"**은 지금 시장에 없습니다. "비슷한 조건의 학생 50명 중 43명이 이 선생님과 3개월 이상 수업을 유지했습니다" — 이런 말을 데이터로 할 수 있는 곳이 없습니다. 이건 데이터가 있어야만 가능하고, 데이터는 시간이 지날수록 쌓입니다. 경쟁자가 돈으로 살 수 없는 자산입니다.

### 5월이 데이터를 쌓았고, 6월은 그걸 쓴다

5월 Phase 1에서 체크인 응답, 코칭 리포트, 프로필 입력률 개선, taxonomy 시드가 돌아가기 시작했습니다. 6월은 이 데이터를 실제 학부모 구매 결정면에 반영하여 **전환율을 직접 올리는** 단계입니다.

## 2. 공고 등록 즉시 AI 매칭 추천

### 현재: 학부모가 직접 찾아야 한다

학부모가 공고를 등록하면, 선생님 목록이 쏟아집니다. 학부모는 이 목록을 하나하나 훑으면서 "이 사람이 우리 애한테 맞을까?"를 스스로 판단해야 합니다. 판단 기준도 없고, 비교 근거도 없습니다. 결국 프로필 사진과 자기소개 글 몇 줄로 결정합니다.

### 6월: 공고 등록 즉시 "이 선생님이 맞는 이유"를 보여준다

공고가 등록되는 순간, AI가 김과외 49만 건 매칭 이력과 12만 건 리뷰 데이터를 분석하여 **추천 이유 카드**와 함께 선생님을 제안합니다. 단순히 "추천 1위"가 아니라, **"비슷한 조건의 학생 중 87%가 이 선생님과 3개월 이상 수업을 유지했습니다"** 같은 구체적 근거를 제시합니다.

핵심 구성요소는 세 가지입니다.

- **자연어 검색 확장**: 3월에 도입한 AI 자연어 검색을 공고 등록 흐름에 통합합니다. 학부모가 적은 요구사항 문장을 그대로 분석하여 적합한 선생님을 찾습니다.
- **추천 이유 카드**: "왜 이 선생님인지"를 데이터 기반으로 설명합니다. 유지율, 리뷰 키워드, 유사 학생 매칭 성공률 등.
- **플레이빙 조합**: 선생님의 수업 스타일과 학생의 학습 성향을 매칭합니다. 5월에 축적된 체크인 데이터와 코칭 리포트가 여기서 활용됩니다.

**매칭 시간 목표: 며칠 → 즉시.** 학부모가 공고를 올리고 기다리는 시간을 없앱니다.

## 3. 이상 징후 감지 v1

### 수업이 깨지기 전에 잡는다

과외가 중단되는 건 갑자기 일어나지 않습니다. 징후가 먼저 나타납니다. 첫 수업 후 연락이 없거나, 결제가 지연되거나, 체크인 만족도가 낮게 나오거나. 지금은 이 신호를 아무도 보지 않습니다. 학부모가 조용히 떠나면 그대로 이탈입니다.

### 6월: 세 가지 이상 신호를 자동으로 잡는다

5월에 도입한 매주 체크인 시스템에서 데이터가 쌓이기 시작했습니다. 6월은 이 데이터에서 **이상 징후 세 가지**를 자동 감지합니다.

1. **첫 수업 후 미연락**: 시범 수업이 끝났는데 72시간 내 후속 연락이 없으면 알림을 보냅니다. 이 구간에서 이탈률이 가장 높습니다.
2. **결제 지연**: 정규 수업 전환 후 결제 예정일을 넘기면 선생님과 학부모 양쪽에 부드러운 리마인더를 발송합니다.
3. **체크인 만족도 저조**: 체크인 응답에서 만족도 1~2점이 연속 2회 이상 나오면, 중재 개입 트리거로 연결합니다.

감지만 하는 것이 아니라, 각 신호에 대응하는 **자동 액션**을 설계합니다. 알림 발송, 리마인더, 중재 연결까지 사람 개입 없이 1차 대응이 돌아가는 구조를 만듭니다.

## 4. SEO 프로그래매틱 랜딩

### 검색에서 김과외를 만나게 한다

학부모의 과외 탐색은 대부분 네이버/구글 검색에서 시작됩니다. "강남 수학 과외", "중3 영어 과외 추천" 같은 검색어입니다. 지금 김과외는 이 검색 결과에 거의 노출되지 않습니다. 4월에 SEO 프로그래매틱 랜딩의 기초를 설계했고, 6월에 본격 실행합니다.

### 10만+ 페이지, AI가 보강한 프로필

지역 × 과목 × 학년 조합으로 **10만 페이지 이상**의 랜딩 페이지를 자동 생성합니다. 각 페이지에는 해당 조건에 맞는 선생님 프로필이 들어가되, 단순 나열이 아니라 AI가 보강한 정보를 포함합니다.

- 해당 지역/과목에서의 매칭 성공률
- 유사 조건 학생들의 리뷰 요약
- 추천 이유 카드 (§2와 동일한 데이터 기반)

**선행 조건**: 4월 SEO 기초 작업 완료 + 5월 프로필 구조화 데이터 축적이 전제입니다. 이 두 가지가 갖춰져야 랜딩 페이지의 콘텐츠 품질이 보장됩니다.

## 5. 지역 기반 추천 고도화

대면 과외는 결국 거리가 핵심입니다. 아무리 좋은 선생님이어도 편도 1시간이면 수업이 오래 가지 않습니다. 지금 김과외의 지역 필터는 단순 행정구역 기반입니다.

6월에는 **실제 이동 거리 기반 추천**으로 전환합니다. 선생님과 학생 간 실제 통학 거리를 계산하고, 매칭 추천에 거리 가중치를 적용합니다. "우리 동네에서 수업 가능한 선생님" 순서로 보여주되, §2의 추천 이유 카드와 결합하여 "가깝고 + 잘 맞는" 선생님을 상위에 노출합니다.

## 6. Phase 전환 기준과 이후 방향

### 5월 → 6월 전환 조건

Phase 1(5월)에서 Phase 2(6월)로 넘어오는 기준은 다음과 같습니다.

- 매주 체크인 응답률 30% 이상 안정화
- 코칭 리포트 최소 1,000건 축적
- 프로필 구조화 입력률 개선 확인
- taxonomy 시드 정상 동작 확인

### 6월 종료 시 달성 목표

- AI 즉시 매칭 추천 → 공고 등록 후 24시간 내 매칭률 개선
- 이상 징후 감지 → 이탈 사전 포착률 측정 시작
- SEO 랜딩 → 검색 유입 트래픽 증가 추이 확인
- 지역 추천 → 대면 매칭의 거리 기반 만족도 측정

### 7월 이후: Phase 3 — 교육 관계 운영 플랫폼

6월까지 "판단 보조 시스템"이 안정되면, 7월부터는 **교육 관계 운영 플랫폼(교육 OS)**으로 확장합니다. 매칭 이후의 수업 관계 전체를 플랫폼이 운영하는 단계입니다. AI 에이전트가 수업 일정 조율, 커리큘럼 제안, 진도 추적까지 수행하는 구조를 설계합니다. Phase 3는 7월 로드맵에서 상세히 다룹니다.

## 7. 매출 레버 요약

| 레버 | 기대 효과 |
|------|-----------|
| AI 즉시 매칭 추천 | 전환율 직접 개선 — 판단 부담 해소로 결제 전환 ↑ |
| 이상 징후 감지 | 이탈률 감소 — 깨지기 전에 잡아서 LTV ↑ |
| SEO 프로그래매틱 랜딩 | 신규 유입 ↑ — 검색 노출로 CAC ↓ |
| 지역 기반 추천 | 매칭 품질 ↑ — 거리 불만 이탈 감소 |

5월이 데이터를 쌓는 달이었다면, **6월은 그 데이터로 돈을 버는 달**입니다.
