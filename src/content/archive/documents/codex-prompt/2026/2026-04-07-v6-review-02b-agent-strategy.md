## 1줄 요약

에이전트 시대 대비 전략 관점에서의 코덱스 검토 프롬프트.

## 핵심 결론

- AI 에이전트 시대 김과외 포지셔닝 검토
- 대체불가능 자산 식별 요청
- 에이전트 매칭 v1/v2 매출 기여도 분석

## 전문

# 코덱스 검토 요청 — 관점 2: 에이전트 시대 대비 전략

## 역할

너는 AI 에이전트 생태계와 플랫폼 전략에 정통한 기술 전략 검토자야. 아래 로드맵을 읽고, **"에이전트 시대 대비 개편"이라는 축 2의 전략이 실제로 경쟁 우위로 작동하는지** 검증해줘.

## 검토 범위

리소스 현실성(인력, 일정, 예산)은 검토하지 마. **에이전트 시대에 김과외가 살아남는 전략으로서의 타당성**만 봐줘.

## 배경

김과외는 과외 매칭 플랫폼이다. 등록 선생님 65만명, 누적 매칭 49만건, 누적 리뷰 12.5만건. 연매출 약 81억. 경쟁자는 숨고, 탈잉, 크몽 등.

## 검토 질문

1. **데이터 구조화의 경쟁 우위:** 프로필·설문·taxonomy를 구조화하면 "에이전트가 김과외를 신뢰할 수 있는 매칭 서비스로 인식한다"고 주장하는데, 이 구조화가 숨고·탈잉 등 경쟁자 대비 진짜 차별점이 되는가? 경쟁자도 같은 작업을 할 수 있지 않은가? 김과외만의 데이터 해자(moat)는 무엇인가?

2. **에이전트 타이밍:** 로드맵은 Phase 3(7~8월 이후)에 에이전트 API/MCP를 열겠다고 하는데, 2026년 하반기에 과외 매칭 영역에서 에이전트 트래픽이 유의미한 수준으로 올까? 너무 이른 투자는 아닌가, 아니면 선점해야 하는가? 에이전트 시대의 현실적 타임라인은?

3. **MCP/API 전략의 구체성:** MCP 서버 배포, Registry 등록, ChatGPT 앱 디렉토리 등록을 언급하는데, 각각의 현실적 진입 장벽은? 과외 매칭처럼 니치한 버티컬에서 에이전트가 직접 API를 호출하는 시나리오가 현실적인가, 아니면 에이전트가 웹 검색→김과외 사이트 방문 경로가 더 현실적인가?

4. **에이전트가 선택하는 기준:** 에이전트가 여러 과외 플랫폼 중 김과외를 선택하는 판단 기준이 무엇일지에 대한 가설이 충분한가? 구조화된 데이터 외에, 가격 경쟁력, 응답 속도, 매칭 성공률, API 안정성 등 에이전트가 보는 다른 요소는?

5. **"두 축이 서로를 강화한다"는 주장:** 재이용률(축1)과 에이전트 대비(축2)가 상호 강화한다고 하는데, 이 연결이 정말 강한가? 설문 데이터가 에이전트 판단 근거가 된다는 주장은 타당한가? 에이전트가 실제로 참고할 만한 수준의 데이터가 얼마나 쌓여야 하는가?

6. **놓치고 있는 에이전트 시대 위협:** 에이전트 시대에 김과외가 직면할 수 있는 위협(중개 플랫폼 건너뛰기, 에이전트가 직접 선생님 DB 구축, 대형 플랫폼의 과외 영역 진출 등)에 대한 방어 전략은 충분한가?

## 출력 형식

각 질문에 대해: (1) 전략적으로 타당한 부분, (2) 가설이 약한 부분 또는 과대평가, (3) 보완 제안을 구분하여 답변해줘.
