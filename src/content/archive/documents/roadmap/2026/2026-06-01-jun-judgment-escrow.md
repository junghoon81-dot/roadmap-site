## 1. 왜 "판단 시스템"인가

3월에 소개서를 구조화하고, 4월에 SEO와 매출 장치를 붙이고, 5월에 체크인으로 데이터를 쌓기 시작했습니다. 이 모든 것의 목적지가 6월 판단 시스템입니다.

### 학부모가 사는 건 "선생님 목록"이 아니다

학부모가 김과외에 돈을 내는 이유를 정확히 짚어야 합니다.

학부모는 선생님 프로필을 구경하려고 오는 게 아닙니다. 학부모가 진짜 사는 건 **"우리 아이에게 이 선생님이 맞을 거라는 확신"** 입니다. 과외 선생님을 고르는 건 학부모 입장에서 상당히 불안한 의사결정입니다. 잘못 고르면 아이의 시간과 돈을 동시에 날립니다. 학원은 이름이라도 있고 커리큘럼이라도 정해져 있는데, 과외는 선생님 한 명에 전적으로 의존합니다. 그래서 학부모는 "가장 좋은 선생님"보다 **"실패하지 않을 선생님"** 을 찾습니다.

지금 김과외가 하고 있는 건 "선생님 프로필을 보여주고, 학부모가 알아서 판단하게 하는 것"입니다. 이건 본질적으로 판단의 부담을 학부모에게 떠넘기는 구조입니다. 프로필을 아무리 예쁘게 만들어도, 학부모는 "이 사람이 진짜 우리 애한테 맞을까?"라는 불안을 혼자 안고 결정해야 합니다.

### "선생님 목록" vs "판단 근거"의 돈 차이

"선생님 목록을 보여주는 곳"은 대체제가 많습니다. 숨고, 크몽, 당근, 네이버 카페. 프로필 DB는 누구나 만들 수 있고, 선생님은 여러 플랫폼에 동시 등록합니다. 이 시장에서 경쟁하면 결국 광고비 싸움, 수수료 인하 싸움으로 갑니다.

반면 **"왜 이 선생님이 우리 아이에게 맞는지 근거를 대주는 곳"** 은 지금 시장에 없습니다. "비슷한 조건의 학생 50명 중 43명이 이 선생님과 3개월 이상 수업을 유지했습니다" — 이런 말을 데이터로 할 수 있는 곳이 없습니다. 이건 데이터가 있어야만 가능하고, 데이터는 시간이 지날수록 쌓입니다. 경쟁자가 돈으로 살 수 없는 자산입니다.

### 이 구조가 미래에도 그대로 통하는 이유

판단 근거 데이터를 쌓는 건, 지금 당장은 학부모가 "이 선생님 괜찮겠다"는 확신을 빨리 갖게 해서 전환율을 올리는 데 쓰입니다.

그런데 이 데이터 구조는 AI 시대에도 그대로 유효합니다. 가까운 미래에 AI 에이전트가 여러 플랫폼을 동시에 탐색해서 비교해주는 건 시간문제입니다. 그때가 오면 "선생님 프로필을 보여주는 것" 자체의 가치는 줄어듭니다. 검색은 AI가 다 해주니까요. 그 상황에서 살아남는 건, **AI가 읽어도 의미 있는 판단 근거를 가진 곳** 입니다.

지금 전환율을 올리기 위해 하는 일이 미래 대비까지 겸하는 구조입니다. 설문으로 만족도를 수집하면 지금은 재매칭 유도에 쓰이고, 나중에는 AI 에이전트가 읽는 신뢰 근거가 됩니다. 추천 이유 카드는 지금은 학부모가 보는 것이고, 나중에는 에이전트가 파싱하는 것입니다. taxonomy로 실패 사유를 분류하면 지금은 매칭 정밀도 개선에 쓰이고, 나중에는 에이전트에게 "이 플랫폼은 이런 기준으로 매칭한다"는 구조화된 정책이 됩니다.

### 김과외의 미래 포지션

김과외의 본질 재정의: **"과외 선생님을 찾는 곳" → "부모가 교육 판단을 더 안전하고 덜 불안하게 위임할 수 있는 시스템"**

학부모가 실제로 물어보는 건 단순히 "수학 선생님 누구?"가 아닙니다. 우리 애 성향에 맞는가, 3개월 안에 성과 가능성이 있는가, 숙제 관리까지 되는가, 비용 대비 효율이 맞는가, 지금 갈아타야 하는가 — 이런 연속 판단입니다. 여기서 김과외가 잘하면, 그냥 매칭앱이 아니라 교육 의사결정의 기본 인터페이스가 될 수 있습니다.

### 이 문서의 성격

이 문서는 "확정된 계획"이 아니라 **"논의를 위한 프레임"** 입니다. 팀 안에서 나오고 있는 여러 아이디어들을 하나의 프레임 안에서 연결하고 우선순위를 논의하자는 겁니다.

## 2. 이 로드맵이 어떻게 돈을 더 버는가

김과외의 매출 공식은 단순합니다:

**매출 = 유입 x 성사율 x 건당 수수료 x 재이용률**

이 4가지 레버를 기준으로, 6월의 각 기능이 어디를 움직이는지 따져봅니다.

### 레버 1: 성사율 향상 (①②③④가 여기에 걸림)

현재 구조: 학부모가 공고를 올리면 → 선생님이 지원하기를 기다림 → 학부모가 프로필을 보고 직접 판단. 이 과정에서 "프로필만 봐서는 판단이 안 되니까 그냥 넘기는" 케이스가 상당수일 겁니다.

**설문요청 기능(①)**: 수업 결과 데이터가 쌓이면 추천 이유 카드에 "비슷한 조건 학생 만족도 4.3/5.0" 같은 실제 근거를 붙일 수 있습니다. 동시에 설문 완료 후 "다른 과목도 찾고 계신가요?" 재매칭 유도를 붙이면 재이용률(레버 2)도 함께 올라갑니다.

**추천 이유 카드 고도화(②)**: 4월 패치에 이미 v1이 있고, 6월에 설문 데이터 기반 실제 수업 성과를 추가합니다.

**프로필 입력률(③)**: 빈 필드가 채워지면 자연어검색의 매칭 재료가 풍부해지고 추천 이유 카드의 설명력도 올라갑니다.

**플레이빙 강화(④)**: 학부모 세부 니즈 기반으로 매칭 정밀도가 올라가면, "맞는 선생님이 없어서 그냥 마감"하는 공고가 줄어듭니다.

### 레버 2: 재이용률 향상 + 누락 매출 회수 (①⑥이 여기에 걸림)

현재 구조: 수업이 성사되면 → 김과외와의 관계가 사실상 끝남. 카톡으로 빠져나간 성사 누락, 시범 후 미전환, 90일+ 휴면 학부모 등 이미 만들어진 매출 기회가 새고 있습니다.

**설문요청 기능(①)**: 수업 중인 학부모에게 정기적으로 알림톡을 보내면, 김과외가 "매칭 후에도 케어하는 곳"이라는 인식이 생깁니다. 1회 성사 후 이탈하는 학부모가 월 약 708명인데, 이 학부모들의 재이용을 유도하면 추가 광고비 없이 기존 고객에서 매출을 만드는 구조입니다.

### 레버 3: 이탈 감소 (⑤가 여기에 걸림)

**매칭 taxonomy(⑤)**: 수업 중 이탈 사유가 구조화되면, 패턴이 보이기 시작합니다. "예산 불일치"가 이탈 1위 사유라면 매칭 시점에 예산 필터를 더 정밀하게 적용하면 됩니다. taxonomy는 매칭 알고리즘의 피드백 루프입니다.

### 레버 4: 유입 확대는 이 기반 위에서 더 효과적

판단 시스템이 먼저 깔리면: 성사율이 오르고 → 이후 SEO로 유입을 늘렸을 때 → 높아진 성사율로 전환하니까 → 같은 유입에서 더 많은 매출이 나옵니다. 전환율 개선이 먼저 되어 있으면, 유입을 늘렸을 때 효과가 곱해지는 구조입니다.

### 돈의 흐름 요약

| 기능 | 움직이는 레버 | 기대하는 방향 |
|------|-------------|-------------|
| ① 설문요청 | 성사율 ↑, 재이용률 ↑ | 판단 데이터 축적 → 추천 정밀도 향상 + 재매칭 유도 |
| ② 추천 이유 카드 고도화 | 성사율 ↑ | 프로필 조회 → 문의 전환율 개선 |
| ③ 프로필 입력률 끌어올리기 | 성사율 ↑ | 기존 빈 필드 채우기 → 매칭 재료 풍부화 |

## 4. 핵심 기능 7개

### ① 설문요청 기능

②–⑤에서 활용할 판단 데이터의 시작점입니다.

**발송 타이밍:**

- 수업 시작 후 2주: "수업은 잘 진행되고 있나요?" (만족도 + 선생님 강점)
- 수업 시작 후 1개월: "한 달간 수업 어떠셨어요?" (만족도 + 지속의향 + 보완점)
- 수업 시작 후 2개월: "수업 성과는 어떠세요?" (목표 달성도 + 추천 의향)
- 수업 종료 시점: "어떤 이유로 수업을 마무리하시나요?" (종료 사유)

**발송 방식:** 카카오 알림톡 (4월 패치 납부요청 인프라 재활용). 알림톡 클릭 → 자동 로그인 → 간편 응답 화면 (탭 3–4번이면 완료).

**수집 항목:**

- 수업 만족도 (1–5)
- 선생님 강점 (복수선택: 숙제관리, 개념설명, 동기부여, 시험대비, 진도관리, 커뮤니케이션)
- 수업 지속 의향 (예 / 아니오 / 고민중)
- 지속하지 않는 경우 사유 (복수선택: 목표달성, 비용, 선생님불만족, 시간, 기타)
- 자유 의견 (선택)

**설문 완료 후 플로우:**

- 만족도 높음 (4–5): "다른 과목도 찾고 계신가요?" → 재매칭 유도
- 만족도 보통 (3): "어떤 부분이 아쉬우신가요?" → 추가 피드백 수집
- 만족도 낮음 (1–2): "더 잘 맞는 선생님을 찾아드릴까요?" → 재매칭 제안

설문 자체가 **재이용 유도 채널** 이 됩니다. 월간 성사 건수가 수천 건이라면, 3개월 후면 만 건 이상의 구조화된 매칭 성과 데이터가 확보됩니다.

### ② 추천 이유 설명 카드 고도화

4월 패치에 이미 v1이 들어가 있습니다. 선생님 상세페이지의 "김과외 AI가 확인한 내용" 섹션에서 학년 일치 ✅, 과목 일치 ✅, 수업스타일 일치 ✅ 같은 조건별 매칭 이유가 표시되고, 검색 결과에 "AI 추천도 100%"와 조건 매칭 태그가 나옵니다.

6월에는 이 기반 위에 **설문 데이터 기반의 실제 수업 성과** 를 추가하여 고도화합니다.

- **현재 (v1)**: 프로필 정보 기반 조건 매칭 — "학년 일치", "과목 일치", "수업스타일 일치"
- **6월 고도화**: 실제 매칭 결과 기반 근거 추가 — "비슷한 조건 학생 47명 기준 만족도 4.3/5.0", "장기 유지율이 높은 선생님"

기술적으로는 4월의 자연어검색 API 적합도 스코어 + ①에서 수집된 설문 데이터 + ⑤ taxonomy 분류 체계를 조합하여, 규칙 기반 템플릿에서 점차 실제 데이터 기반 이유로 전환합니다.

### ③ 선생님 프로필 입력률 끌어올리기

핵심은 새 컬럼을 추가하는 게 아니라 **기존 컬럼의 입력률을 끌어올리는 것** 입니다.

**입력률을 올리는 방법:**

- 공고 지원 시점 유도: 선생님이 공고에 지원할 때 "이 정보를 입력하면 학부모에게 추천될 확률이 올라갑니다" 메시지와 함께 미입력 항목을 보여줌
- 선생님 부스트 팝업과 연계: 앱 접속 시 "프로필 완성도 70% → 90%로 올리면 공고 노출이 올라갑니다" 넛지
- 프로필 완성도 시각화: 선생님 마이페이지에 진행률 바를 추가, 빈 항목을 명시하고 바로 입력으로 이동

**특히 채워야 할 핵심 필드** (자연어검색 매칭 정밀도에 직접 영향):

- `student_type_confident_grade` / `student_type_weak_point`: 자신있는 학생 수준 및 약점 → 학부모 공고의 tutee_level, specialty_point와 매칭
- `homework_assignment_method` / `feedback_cycle` / `feedback_method`: 수업 운영 방식 → 추천 이유 카드에 "이 선생님은 매 수업 후 숙제 출제 + 매주 피드백" 표시 가능
- `specialty_point`: 과목별 세부파트 → 학부모의 "보완필요 파트"와 직접 매칭

### ④ 학부모 플레이빙 강화

학부모 공고에도 DB 컬럼은 이미 꽤 있습니다. tutee_level(학생 수준), tutee_goal(학습 목표), preferred_tutor_gender(선호 성별), specialty_point(보완필요 파트) 등.

**학부모 공고 317,154건 중 입력률:**

| 필드 | 입력률 |
|------|--------|
| 선호 성격 태그 (personality) | 59.6% (188,925건) — 유일하게 높음 |
| 학생 수준 (tutee_level) | 9.0% (28,454건) |
| 학습 목표 (tutee_goal) | 거의 0건 (4월 패치 신규 필드) |
| 보완필요 파트 (specialty_point) | 2건 |

**과제 1: 4월 패치 신규 필드의 입력률 올리기.** tutee_goal, specialty_point 등이 실제로 채워지도록 공고 등록 UX를 보완합니다.

**과제 2: 아직 없는 필드 추가.**

- 관리 강도 선호: 자율형 / 관리형 / 강한관리형 — 같은 "중2 수학"이라도 숙제관리가 필요한 학생과 자기주도가 되는 학생은 다른 선생님이 맞습니다.
- 이전 과외 경험: 처음 / 있음–만족 / 있음–불만족 — 불만족 사유를 함께 받으면 같은 실수를 반복하지 않는 매칭이 가능합니다.

**미성사 공고 재활용:** 기존에 마감된 미성사 공고의 학부모에게 "이 정보만 추가해주시면 더 잘 맞는 선생님 찾아드릴게요" 알림톡. 유입 비용 0원으로 추가 매칭을 시도합니다.

### ⑤ 매칭 실패/성공 이유 Taxonomy

①–④가 "데이터를 수집하는 도구"라면, ⑤는 "수집된 데이터를 분류하는 체계"입니다. 체계 없이 데이터만 쌓으면 분석이 안 됩니다.

**갈래 1: 앞으로 쌓는 데이터 — 서비스 내 3개 시점 선택지**

시점 1 — 선생님 제안 거절 시: 경력/학력 부족, 예산 불일치, 지역 멀음, 시간대 불일치, 프로필 부족으로 판단 어려움, 그냥 넘김

시점 2 — 공고 마감 시: 좋은 선생님을 찾았음, 맞는 선생님이 없었음, 다른 곳에서 찾았음, 과외를 안 하기로 함

시점 3 — 설문요청(①)에서 수업 종료 사유: 목표 달성, 비용 부담, 선생님 불만족, 시간 불일치, 학원으로 전환

이 3개가 합쳐지면 **매칭 전(제안 거절) → 매칭 종료(공고 마감) → 수업 중/후(설문)** 전 구간의 판단 데이터가 자동으로 쌓입니다.

**갈래 2: 기존 데이터 영끌**

설문이 쌓이기를 기다릴 필요 없습니다. 지금 김과외 안에 이미 흩어져 있는 데이터를 전부 끌어모읍니다.

- **고객센터 접수 이력**: 학부모가 특정 선생님에 대한 불만을 접수한 기록. CS 유형별로 분류하면 이것 자체가 이탈 원인 taxonomy의 시드 데이터입니다.
- **앱 내 후기/평점**: 특히 낮은 평점 후기의 키워드를 분류하면 불만족 사유 패턴이 보입니다.
- **매칭 이력 통계**: 특정 선생님의 성사율, 평균 수업 유지 기간, 중도 이탈률.
- **학부모 전화 설문**: 알림톡 설문 응답률이 낮을 수 있습니다. 그러면 기다리지 말고, 운영팀이 직접 학부모에게 전화를 돌려서 매칭 결과를 수집합니다. 초기 데이터 100–200건을 빠르게 확보하는 게 중요합니다.
- **고객센터 기록 재분류**: 기존 CS 기록을 taxonomy 선택지 기준으로 소급 태깅합니다.

요점은: 설문 시스템이 완성되어 데이터가 자동으로 쌓이기를 기다리는 게 아니라, 지금 당장 가능한 모든 방법으로 매칭–결과 데이터를 끌어모은다는 것입니다.

### ⑥ 소소한 매출 향상 포인트 모음

①–⑤가 판단 시스템의 기반을 쌓는 작업이라면, ⑥은 지금 당장 매출에 붙는 실무 장치들입니다.

**A. 성사 누락 잡기 (학부모 진전문의 알림톡):** 채팅까지 했는데 성사등록 없이 카톡으로 빠지는 학부모를 잡습니다. 성사등록 회피로 정지된 유저 중 55%가 인앱 시그널 제로 — 외부에서 수업을 시작하고 성사등록을 안 하는 케이스입니다. 채팅 시작 후 7일 경과 + 성사등록 없음 → 알림톡 발송 → 웹뷰 체크리스트에서 수업시작 여부 확인.

**B. 시범→정규 전환 넛지:** 시범수업 후 정규 전환하지 않는 건수: 무료 10,284건 + 유료 20,924건 = 총 31,208건. 시범수업 완료 후 14일 경과 → 알림톡 발송 → 정규 전환 유도.

**C. FOMO 성사 알림:** 고민 중인 선생님이 다른 학부모와 성사되면 즉시 푸시. 대상: 63,668명 (모집중 공고 + 채팅/찜 선생님 있는 학부모).

**D. 리뷰 도착 알림:** 채팅한 선생님에게 새 만점 후기가 달리면 학부모에게 푸시. 후기는 의사결정에 가장 강한 트리거인데 현재 활용이 안 되고 있습니다.

**E. 공고 이탈 복구:** 공고 작성 중 이탈 24시간 후 푸시 → 폼 복원. 4월 패치에서 스텝을 7→6으로 줄였기 때문에 복원 시 완료 확률이 이전보다 높을 것으로 예상됩니다.

**F. 선생님 부스트 (과목 확장 + 화상과외 추가):** 공고당 제안 6건 미만인 "공급 사막" 과목이 다수 (요가 0.6건, 테니스 1.0건, 마케팅 1.2건 등). 반면 수학 I/II는 93건으로 과포화. 실제 가르칠 수 있는 선생님이 DB에 있지만 해당 과목을 등록하지 않은 "잠자는 선생님"이 85–90%. 앱 접속 시 인앱 팝업으로 화상과외 추가 + 추천과목 추가를 유도합니다.

**G. 휴면 학부모 깨우기:** 90일 이상 미활동 학부모 1,594,664명. 이 중 성사 이력 있는 학부모만 215,951명. 이전 공고 조건에 맞는 선생님 리스트를 알림톡으로 발송. 주 10–20만씩 롤링, 1인당 1회만.

**H. 수업 결과 설문:** ①의 설문요청과 맞물리는 기능입니다. 성사 3개월 후 알림톡 → 분기형 설문(수업 상태 → 수업 기간 → 종료 이유 → 성적 변화 → 후기 유도). 완료 시 스타벅스 아메리카노 지급 (월 약 900명 x 4,500원 ≈ 400만원/월).

### ⑦ 안심결제 전국 도입 + 요금 역전 해소

**현재 구조:** 학부모가 공고를 등록할 때 결제방식을 선택합니다 — "선생님에게 직접결제" 또는 "안심결제". 안심결제를 선택하면 김과외를 통해 결제하고, 수업 완료 확인 후 선생님에게 Toss 서브몰로 정산됩니다. 한 번 설정된 결제방식은 공고 수정으로 변경할 수 없습니다.

**현재 제한:** 안심결제 옵션은 **서울 지역(`cityId === 1`) 공고에만** 노출됩니다. 서울 외 지역 학부모는 공고 등록 시 안심결제 선택지 자체를 볼 수 없습니다.

**6월: 전국 확대 + 요금 역전 해소**

`hasEscrowAccess` 조건에서 `cityId === 1` 제한을 제거하여 전국 모든 지역에서 안심결제를 선택할 수 있게 합니다. 동시에 일부 구간에서 발생하는 요금 역전 현상(안심결제 이용 시 오히려 비용이 높아지는 케이스)을 해소합니다. 안심결제가 학부모에게 "비싸지 않으면서 안전한" 선택지가 되어야 전국 확대가 의미 있습니다.

**판단 시스템과의 연결:** 안심결제 비율이 올라갈수록 ① 설문요청의 발송 대상도 정확해집니다. 결제가 김과외 안에서 이루어지면 "이 학부모가 수업 중인지"를 확실히 알 수 있으니까요. 매칭부터 결제까지 김과외 안에서 완결되는 구조입니다.

## 5. 7월 이후 — 6월이 여는 길

6월까지가 "기반 + 데이터 + 판단 시스템"이었다면, 7월부터는 이걸 실제 매칭 경험으로 바꿉니다.

### 7월: AI 즉시 매칭 (성수기)

학부모가 공고를 올리면 → 자연어검색 API가 내부적으로 호출 → "이 공고에 가장 적합한 선생님 3명"을 즉시 추천 + 추천 이유 카드. 3월 자연어검색 API + 5월 프로필 데이터 + 6월 추천이유카드 + taxonomy가 합쳐져서 가능해지는 기능입니다. 성수기(25년 7월 매출 9.72억)에 맞춰서 최대 효과를 노립니다.

### 8~9월: 시스템 고도화

체크인 데이터 + taxonomy 피드백 루프로 매칭 알고리즘을 계속 개선합니다. 코칭리포트 AI 자동화, 선생님 등급제, 이상징후 감지 v2, 교육 관계 대시보드.

### 10~11월: AI 에이전트 매칭

학부모가 대화형으로 조건을 말하면 AI가 파악해서 추천하는 수준까지 올립니다. 이 시점에 체크인 5개월치 + 설문 4개월치 + taxonomy 분류 데이터가 충분히 쌓여있으므로 가능해집니다. 이것이 6월 문서 §1에서 말한 "에이전트 시대 대비"의 실체입니다.

### 12월: 교육 OS v1

매칭~수업진행~종료까지 전 과정이 김과외 안에서 완결되는 구조를 완성합니다. 학부모 1명의 생애가치(LTV)가 "수수료 1회"에서 "수수료 1회 + 과목 확장 + 프리미엄 운영 요금"으로 바뀝니다.

## 6. 요약

| 기능 | 매출 레버 |
|------|-----------|
| ① 설문요청 | 판단 데이터 축적 → 추천 정밀도 + 재매칭 유도 |
| ② 추천 이유 카드 | 프로필 조회 → 문의 전환율 ↑ |
| ③ 프로필 입력률 | 매칭 재료 풍부화 |
