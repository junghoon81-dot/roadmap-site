## 1줄 요약

전략 분석 아카이브의 정보구조 설계부터 코워크 스킬 자동화까지 6개 질문으로 구성된 코덱스 감사 프롬프트입니다.

## 핵심 결론

- 단일 md 아카이브 → 다중 md + generated index JSON 2계층 전환 설계
- 기존 전략 파일 30개+ 마이그레이션 매핑 및 3-Phase 실행 계획
- 코워크 스킬 자동 저장/로드 모드 설계 및 선택적 컨텍스트 로딩 구조

## 전문

# Codex 감사 프롬프트 — 전략 아카이브 시스템 설계 + 코워크 스킬 등록

---

## 배경

김과외 로드맵 사이트(https://roadmap-site-nu.vercel.app)에 **아카이브 탭**이 있다. 현재 `src/content/archive/작업_아카이브.md` 단일 파일(132KB, 1,869줄)에 로드맵 md 원문 4개 + Codex 자문 6개 + 교차 분석 1개가 저장되어 있다.

**문제:** 지금까지 수행한 전략 분석은 이 아카이브에 담긴 것보다 훨씬 많다. Codex 검토 프롬프트(9종), Codex 답변 검토(6종+), 자체 DB 분석(안심결제 갭 분석 등), 5월 로드맵 기획(v1~v9), SOP 문서 등이 흩어져 있으며, 새 분석을 할 때마다 이전 분석 컨텍스트가 유실된다.

**목표:**
1. 흩어진 모든 전략 분석 콘텐츠를 아카이브 페이지에 구조화하여 통합
2. 앞으로 새 분석이 생길 때마다 자동으로 아카이브에 추가되는 코워크 스킬 등록
3. 매 분석 세션에서 아카이브를 컨텍스트로 활용할 수 있는 구조 설계

## 질문 1: 아카이브 정보 구조 설계

현재 아카이브는 시간순 단일 문서(flat markdown)이다. 전략 분석이 30개 이상 쌓이면 이 구조는 실용성이 없다.

아래 콘텐츠 유형별로 **아카이브의 최적 정보 구조**(IA)를 설계해 달라:

| 유형 | 예시 | 현재 수량 |
|------|------|-----------|
| 로드맵 원문 | 3월/4월/5월/6월 로드맵 md | 4개 |
| Codex 자문 프롬프트 | 결제·핀테크, 공급측, 유닛이코노믹스 등 | 9종 |
| Codex 자문 답변 요약 | 6개 관점 교차 분석, 개별 답변 검토 | 10+ |
| 자체 데이터 분석 | 안심결제 갭 분석(DB+자금수지표), 매출 구조 분석 | 1+ (향후 증가) |
| 로드맵 기획 이력 | 5월 패치 v1→v9 변경 이력 | 9개 버전 |
| SOP/운영 문서 | 전수검사, 자체검증, 인수인계, 정리검증 | 12개 |
| 코워크 감사 프롬프트 | 코드 전수 감사, AI Tips 감사 등 | 3+ |

요구사항:
- **검색성:** 특정 주제(예: "안심결제 요율")에 대한 모든 분석을 빠르게 찾을 수 있어야 한다
- **연대기:** 같은 주제의 분석이 시간순으로 추적 가능해야 한다
- **컨텍스트 로딩:** 새 분석 시작 시, 관련 과거 분석만 선택적으로 로딩할 수 있어야 한다
- **토큰 효율:** 전체를 한 번에 로딩하면 토큰 낭비. 요약 + 원문 2계층 구조 필요
- **현재 기술 스택:** Next.js + Vercel, 단일 md 파일 기반 렌더링 (서버 컴포넌트), DB 없음

설계에 포함할 것:
1. 디렉토리/파일 구조 (단일 md vs 다중 md vs JSON 인덱스 + md)
2. 각 문서의 메타데이터 스키마 (날짜, 유형, 태그, 관련 문서 링크)
3. 인덱스/목차 자동 생성 방식
4. 요약 계층 (1줄 요약 / 핵심 결론 3줄 / 전문)

## 질문 3: 코워크 스킬 설계

분석이 완료될 때마다 아카이브에 자동 저장하는 코워크 스킬을 설계해 달라.

### 스킬이 해야 할 일

1. **분석 완료 감지:** 코워크 세션에서 전략 분석/Codex 감사/데이터 분석이 완료되면 트리거
2. **메타데이터 추출:** 분석 제목, 날짜, 유형(codex-prompt/codex-review/data-analysis/roadmap/audit), 핵심 태그, 1줄 요약, 핵심 결론 3줄
3. **아카이브 파일 업데이트:** 설계한 구조에 맞게 새 엔트리 추가
4. **인덱스 갱신:** 목차/인덱스 파일 자동 업데이트
5. **Git 커밋 + Vercel 배포:** 변경사항을 자동 커밋하고 Vercel에 반영

### 스킬 구현 제약

- 코워크의 `.claude/skills/` 디렉토리에 `SKILL.md`로 등록
- 스킬은 코워크 세션 내에서 대표님이 "아카이브에 저장해" 또는 분석 완료 후 자동으로 호출
- Mac 로컬의 `~/roadmap-site/` 레포에 직접 파일 추가 + git push
- 서버리스 함수나 외부 API 없이, 순수 파일 조작 + git으로 구현

설계에 포함할 것:
1. `SKILL.md` 전문 (트리거 조건, 실행 단계, 프롬프트)
2. 아카이브 엔트리 템플릿 (새 분석이 추가될 때의 md 포맷)
3. 인덱스 파일 자동 갱신 로직
4. 에러 처리 (git conflict, 빌드 실패 시 롤백)
5. 수동 트리거와 자동 트리거 구분

## 질문 4: 컨텍스트 활용 구조

아카이브에 30개 이상의 분석이 쌓였을 때, 새 분석 세션에서 어떻게 효율적으로 과거 컨텍스트를 활용할지 설계해 달라.

### 시나리오

"안심결제 수수료 구조를 재설계하라"는 지시가 떨어졌을 때:

1. 관련 과거 분석: 안심결제 갭 분석(2026-04-07), Codex 결제·핀테크 자문, Codex 유닛이코노믹스 자문, 5월 로드맵 §6 안심결제 전국 확대
2. 무관한 분석: 체크인 UX 설계, SEO 전략, SOP 문서

이때:
- 전체 아카이브를 로딩하면 토큰 50,000+ 낭비
- 관련 분석 4개만 로딩하면 토큰 5,000~8,000으로 충분

이 "선택적 컨텍스트 로딩"을 어떻게 구현할지:

1. **태그 기반 필터링:** 태그 체계 설계 (예: `escrow`, `revenue`, `codex`, `roadmap`)
2. **요약 계층:** 인덱스에서 1줄 요약으로 스캔 → 관련 문서만 전문 로딩
3. **코워크 스킬 연동:** 분석 시작 시 "관련 아카이브 로드" 스킬이 자동 실행
4. **수동 오버라이드:** 대표님이 "X 분석도 참고해" 하면 추가 로딩

## 질문 5: 웹사이트 UI 개선

현재 아카이브 페이지는 단일 긴 문서를 그대로 렌더링한다. 30개 이상의 분석이 쌓이면:

1. 페이지 로딩이 느려짐 (132KB → 500KB+)
2. 원하는 분석을 찾기 어려움
3. 유형별/시간순 필터링 불가

아카이브 페이지의 UI를 어떻게 개선할지:

1. **카드 그리드 vs 목록 vs 트리:** 어떤 레이아웃이 최적인지
2. **필터/검색:** 유형별, 태그별, 날짜별 필터 UI
3. **상세 보기:** 카드 클릭 → 전문 펼침 or 별도 페이지
4. **기술 구현:** 단일 md 유지 vs 다중 md + 동적 라우팅 vs JSON 인덱스
5. **현재 스택(Next.js 서버 컴포넌트 + Vercel)에서의 제약과 해결책**

## 질문 6: 실행 우선순위

위 1~5를 한번에 다 할 수 없다. 다음 기준으로 우선순위를 정해 달라:

- **즉시 효과:** 다음 분석 세션부터 바로 컨텍스트 활용 가능
- **누적 효과:** 분석이 쌓일수록 가치 증가
- **구현 복잡도:** 코워크 세션 1~2회로 완료 가능한 것부터
- **의존성:** A를 해야 B를 할 수 있는 순서

Phase 1(이번 주) / Phase 2(다음 주) / Phase 3(이후)로 나눠서 구체적 실행 계획을 제시해 달라.

## 컨텍스트: 최근 자체 분석 결과 요약

**안심결제 vs 직접결제 갭 분석 (2026-04-07):**
- 표면 갭 24.3% → 환불/페이백 반영 순매출 갭 20.4%
- 갭 2대 원인: 세션 미정산 45% + 과외비 차이 55%
- 안심결제 장점: 환불 0%, 페이백 0% (건당 3,121원 절감)
- 권장안: 요율 22% + usage_fee 전액 수금 → 갭 12.5%로 축소
- 데이터: Kimstudy/GAWEKIM DB 직접 쿼리 + 월별자금수지표 교차 검증

이 분석이 아카이브에 저장되어야 할 첫 번째 "자체 데이터 분석" 케이스이다.
