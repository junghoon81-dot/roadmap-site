> **요청자:** 대표님 (김과외 CEO)
> **감사 대상:** 코워크 세션 대화기록을 전략 아카이브에 저장하는 구조 설계안
> **배경:** 새 코워크 세션마다 이전 맥락이 유실되어 같은 설명을 반복해야 하는 문제가 빈발함. 대화에서 내려진 의사결정, 확인된 사실관계, 비즈니스 맥락이 아카이브에 없어서 코워크가 아카이브를 읽어도 핵심 정보가 빠져있음.

## 현재 시스템

### 전략 아카이브 (roadmap-site)

디렉토리: `src/content/archive/documents/{type}/{year}/{date}-{slug}.md`
인덱스: `scripts/rebuild-index.mjs` (gray-matter 기반, 스키마 검증, backlinks 자동 생성)
빌드: Next.js SSG, Vercel 배포

현재 문서 타입:
```
roadmap, roadmap-history, codex-prompt, codex-review,
data-analysis, audit, sop, legacy-snapshot
```

현재 문서 수: 22개 (canonical 19, non-canonical 3)

### SKILL.md (Save 모드 10단계)
1. id 검색 → 2. frontmatter → 3. 스키마 검증 → 4. series_id → 5. canonical → 6. 파일 생성 → 7. rebuild-index → 8. 빌드 → 9. diff → 10. 커밋 (push는 대표님 지시 시만)

### 코워크 대화기록 접근 방법

**방법 1: session_info MCP**
- `list_sessions`: 현재 열려있는 세션 목록 조회 (세션ID, 제목, 상태)
- `read_transcript`: 세션ID로 대화 내용 읽기 (limit 파라미터로 메시지 수 제한)
- 제한: 현재 열려있는 세션만 접근 가능. 과거 닫힌 세션은 접근 불가.

**방법 2: 컴팩션 로그 (JSONL)**
- 경로: `/sessions/{session-name}/mnt/.claude/projects/{project-path}/{uuid}.jsonl`
- 컨텍스트 초과 시 이전 대화가 JSONL로 저장됨
- 전체 대화 기록 포함 (user/assistant/tool 메시지)

**방법 3: 수동 복사**
- 대표님이 대화 내용을 복사하여 붙여넣기
- 비효율적이지만 확실한 방법

### 대화기록에서 추출해야 할 정보 유형

1. **의사결정 기록**: "학부모 수수료 0%다", "안심결제 요율 22%로 낮춰라" 등 대표님이 확인/결정한 사실
2. **비즈니스 맥락**: 수수료 구조, 결제 흐름, 운영 현황 등 대표님만 아는 정보
3. **작업 결과**: 코드 수정, 문서 작성, 분석 결과 등 세션에서 수행한 작업
4. **오류 교정**: 아카이브 문서의 잘못된 내용을 대표님이 지적한 기록 (예: "학부모 수수료 없어")

## 설계안 (코덱스 검토 대상)

### 안 1: 새 문서 타입 `session-digest` 추가

```yaml
type: session-digest
artifact_kind: report
```

세션 종료 시 또는 대표님 지시 시, 대화에서 아래 항목을 추출하여 md 파일 생성:

```markdown
---
id: session-digest-2026-04-07-escrow-pricing
title: "안심결제 수수료 구조 확인 및 아카이브 오류 수정"
date: 2026-04-07
type: session-digest
artifact_kind: report
status: final
version: v1
canonical: true
tags:
  - topic/escrow
  - topic/payment
summary_line: "학부모 안심결제 수수료 0% 확인. 코덱스 리뷰의 학부모 4% 가정이 현실 불일치로 아카이브 3개 파일 수정."
key_findings:
  - "학부모 측 안심결제 수수료는 0%. 추가 부과 불가."
  - "안심결제 건당 순매출이 직접결제 대비 20.4% 낮은 구조적 문제 확인"
  - "코덱스 리뷰 문서의 '학부모 4%' 전제가 비현실적임을 확인하고 주석 추가"
related_ids:
  - codex-review-2026-04-07-06-payment-fintech
  - data-analysis-2026-04-07-escrow-gap
series_id: session-digest-2026-04
token_hint: small
---

## 의사결정 기록
- 학부모 안심결제 수수료: **0% 확정** (변경 불가)
- 안심결제의 핵심 문제: 건당 순매출이 직접결제 대비 20.4% 낮음
- 갭 원인: 세션 미정산 45% + 과외비 차이 55%

## 오류 교정
- 핀테크 리뷰 문서(review-06)의 "학부모 측 4% 반영 시" 전제가 현실과 불일치
- 3개 파일에 경고 주석 추가 완료 (커밋 39c4281)

## 미해결 과제
- 학부모 수수료 없이 안심결제 건당 마진 20.4% 갭을 어떻게 메울 것인가
- 반복결제 capture의 실질 수익 구조 파악 필요
```

### 안 2: 기존 문서에 `corrections` 섹션 추가

session-digest를 별도로 만들지 않고, 기존 문서(예: review-06)에 직접 corrections 섹션을 추가하는 방식.

## Corrections (2026-04-07)

## 코덱스에게 묻는 질문 8개

### Q1. 문서 타입 선택
session-digest(안1), corrections(안2), facts(안3) 중 어떤 구조가 가장 효과적인가? 또는 복합 사용이 나은가?

평가 기준:
- 새 세션에서 맥락 복구 속도 (코워크가 읽고 바로 쓸 수 있는가)
- 검색 가능성 (특정 의사결정을 나중에 찾을 수 있는가)
- 유지보수 부담 (문서가 기하급수로 늘어나지 않는가)
- 기존 아카이브 구조와의 정합성

### Q2. 추출 자동화 vs 수동 큐레이션
대화 전문을 아카이브에 넣으면 노이즈가 많고, 핵심만 추출하면 빠뜨리는 게 생긴다.

- 전문 저장 + 요약 생성 (토큰 비용 높음, 누락 위험 낮음)
- 핵심만 추출 (토큰 비용 낮음, 누락 위험 높음)
- 하이브리드: 핵심 추출 + 전문은 별도 경로에 보관

어느 방식이 현실적인가? 코워크 세션당 평균 대화량을 고려하면?
